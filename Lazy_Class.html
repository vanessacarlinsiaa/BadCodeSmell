Lazy Class

Lazy class adalah sebuah class yang tidak memberikan banyak nilai tambah atau fungsionalitas yang cukup signifikan dalam program. Class semacam itu sering kali hanya memiliki beberapa metode, dan tidak memiliki alasan yang kuat untuk keberadaannya.

Alasan utama terjadinya "Lazy CLass":


Class tersebut bisa menjadi beban yang tidak perlu dalam code karena mereka tidak memberikan fungsionalitas yang cukup signifikan untuk membenarkan keberadaannya. Ini mengakibatkan overhead dalam pemeliharaan code dan dapat mempersulit pemahaman terhadap struktur keseluruhan dari program.

Cara Mengatasi

  • Identifikasi class yang memiliki sedikit atau bahkan tidak ada peran yang penting dalam sistem.
  • Mempertimbangkan untuk menggabungkan function class tersebut ke dalam class lain yang terkait
  • Evaluasi kembali kebutuhan program dan pastikan setiap class memberikan kontribusi yang signifikan terhadap function keseluruhan.

Mari kita lihat contoh sederhana di mana sebuah class memiliki "LazyClass". Class ini hanya memiliki satu variabel data dan satu metode untuk memproses data tersebut. Namun, class ini tidak memberikan banyak nilai tambah dalam sistem, sehingga dapat dianggap sebagai contoh dari "lazy class" dalam Dispensable Bad Code Smell.

Sebelum Refactor
                            
    // Product.java
    // Contoh kelas "lazy"
public class LazyClass {
    private int data;

    public LazyClass(int data) {
        this.data = data;
    }

    public void processData() {
        System.out.println("Processing data: " + data);
    }
}
Setelah Refactor
                            
// Product.java
// Penyelesaian: menggabungkan fungsionalitas ke dalam kelas lain atau menghapusnya
// Contoh penyelesaian dengan menggabungkan fungsionalitas ke dalam kelas lain
public class NonLazyClass {
    private int data;

    public NonLazyClass(int data) {
        this.data = data;
    }

    public void processData() {
        System.out.println("Processing data: " + data);
    }
}

Setelah diatasi, kita telah menggabungkan function class "LazyClass" ke dalam class "NonLazyClass". Sekarang, class "NonLazyClass" memiliki variabel data yang sama dan juga metode untuk memproses data. Dengan menggabungkan function ini, kita menghilangkan keberadaan class yang tidak perlu, sehingga mengurangi overhead dalam pemeliharaan code. Langkah ini membantu memperbaiki Dispensable Bad Code Smell dengan menghilangkan class yang tidak memberikan nilai tambah yang signifikan dalam sistem.
